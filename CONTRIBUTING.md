# Contributing Guide

Thank you for your interest in contributing to this project.

This repository is a personal R&D project focused on a reusable geometry processing engine. Contributions are welcome when they align with clean architecture, maintainability, and safe open-source practices.

## Contribution Principles

- Keep the project generic, reusable, and framework-agnostic where practical.
- Preserve modular boundaries and clear separation of concerns.
- Prefer readable, well-tested, and deterministic behavior.
- Keep documentation and tests updated with any meaningful change.

## How to Contribute

1. Fork the repository and create a feature branch.
2. Make focused changes with clear commit messages.
3. Run local validation before opening a pull request:
   - `mvn clean test`
4. Open a pull request with:
   - Problem statement
   - Proposed solution
   - Test evidence
   - Any backward compatibility notes

## Coding and Review Expectations

- Follow existing project style and structure.
- Avoid broad refactors in unrelated areas.
- Include or update tests for behavior changes.
- Update docs when public behavior or contracts change.

## Confidentiality and IP Requirements

Contributors must not submit:

- Proprietary company code
- Confidential data or assets
- Internal system details from employers or clients
- Code copied from restricted repositories

By contributing, you confirm that your submission is your own work or that you have explicit rights to contribute it under the repository license.

## Communication

Please use issues to discuss significant changes before implementation so scope and design can be aligned early.
